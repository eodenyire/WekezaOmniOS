Phase 5 is a major leap in complexity. Up to Phase 4 the system teleports apps by pause → snapshot → transfer → restore. Phase 5 introduces true live migration, where the application keeps running while most of its state is moved to the new node.
This is similar to technologies like VMware vMotion, KVM live migration, and advanced container migration systems.

🎯 Goal

🧠 Core Concept: Pre-Copy Migration
Live migration typically uses a pre-copy algorithm.
Step 1 — Memory Copy
Node A
Running App
   ↓
Copy memory pages to Node B
Step 2 — Dirty Page Tracking
While memory is copied, the app continues running.
Changed pages are marked dirty.
Dirty Pages
   ↓
Re-copy them
Step 3 — Final Sync
Pause app briefly
Copy last changes
Resume on Node B

🧭 migration_controller.py
This coordinates the entire migration.
class MigrationController:

    def __init__(self, streamer, tracker):

        self.streamer = streamer
        self.tracker = tracker

    def start_migration(self, process_id):

        print("Starting live migration")

        # Step 1: initial memory copy
        self.streamer.stream_memory(process_id)

        # Step 2: track dirty pages
        dirty_pages = self.tracker.get_dirty_pages(process_id)

        # Step 3: copy dirty pages
        self.streamer.stream_pages(dirty_pages)

        print("Final sync complete")

🌊 memory_streamer.py
Handles streaming memory pages.
class MemoryStreamer:

    def stream_memory(self, process_id):

        print(f"Streaming memory for process {process_id}")

    def stream_pages(self, pages):

        for page in pages:
            print(f"Transferring page {page}")
In real systems this uses:
•	RDMA
•	high-speed networking
•	memory page copying

🧠 dirty_page_tracker.py
Tracks which memory pages changed.
class DirtyPageTracker:

    def __init__(self):

        self.dirty_pages = []

    def mark_dirty(self, page):

        self.dirty_pages.append(page)

    def get_dirty_pages(self, process_id):

        return self.dirty_pages
Real implementations use:
•	page protection
•	kernel hooks
•	CRIU dirty tracking

📊 page_bitmap.py
Efficiently track memory changes.
class PageBitmap:

    def __init__(self, size):

        self.bitmap = [0] * size

    def mark(self, page):

        self.bitmap[page] = 1

    def get_dirty(self):

        return [i for i, v in enumerate(self.bitmap) if v == 1]
