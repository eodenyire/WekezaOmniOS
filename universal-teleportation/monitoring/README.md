# 📊 WekezaOmniOS: Monitoring Module

The **Monitoring Module** is the "Central Nervous System" of the Universal Application Teleportation (UAT) engine. It provides the observability, health checks, and telemetry required to ensure that every process capture and restoration is executed within safe system parameters.

In **Phase 1**, this module focuses on local system resource tracking and process integrity. In **Phase 2**, it will evolve into a distributed telemetry hub for multi-node cluster health.

---

## 🎯 Purpose & Capabilities

This module ensures that the **WekezaOmniOS** engine remains stable and that teleported applications perform optimally on their new hosts.

* **System Resource Tracking:** Monitor real-time CPU and Memory usage to prevent over-allocation during snapshot creation.
* **Process Health Observation:** Heartbeat checks for "teleported" applications to ensure they have resumed execution correctly.
* **Event Logging & Auditing:** Captures critical lifecycle events (Snapshot Start, Transfer Success, Restoration Complete) for the **Wekeza Bank** audit trail.
* **Node Telemetry:** Prepares data structures to report node health (Online/Offline/Busy) to the `cluster/` management layer.

## 🔄 Integration Workflow

The Monitoring module sits alongside the core teleportation pipeline to provide a safety net:

1. **Pre-Flight:** Checks if the target Node has enough RAM to house the incoming snapshot.
2. **In-Flight:** Observes the `snapshot-engine` to ensure data integrity.
3. **Post-Flight:** Confirms the process is `RUNNING` via the `state-reconstruction` module and triggers a success signal.

## 🛠️ Usage (Phase 1 Simulation)

While we are in the local bootstrapping phase, you can use the monitoring hooks to validate your demo apps:

> **Note:** The monitoring tools will automatically look for PIDs stored in the `temp/` folder generated by your `run_demo.sh` script.

```bash
# Example: Monitor the health of the current teleportation target
python monitoring/health_check.py --pid $(cat temp/demo_app.pid)

```
