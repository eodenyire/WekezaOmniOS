## WekezaOmniOS Documentation Hub

This folder is the canonical entry point for cross-repository documentation.

### What belongs here

- Cross-track architecture and strategy documents.
- Repository-wide plans, roadmaps, and progress summaries.
- Navigation aids for module-level documentation.

### Documentation map

- `REPO_STRUCTURE.md`: Folder ownership rules and naming conventions.
- `../README.md`: Root project overview and implementation tracks.
- `../virtualization-layer/README.md`: Virtualization track docs entry.
- `../interface-emulation/README.md`: Interface emulation track docs entry.
- `../cloud-desktop-model/README.md`: Cloud desktop track docs entry.
- `../universal-teleportation/README.md`: Universal Teleportation track docs entry.
- `../universal-teleportation/docs/INDEX.md`: Detailed map of UAT plans, reports, architecture, and specs.

### Organization status

- Top-level track folders are organized and easy to navigate.
- Universal Teleportation legacy root docs were relocated into categorized `docs/` subfolders.
- Root-level module folders now prioritize implementation code and runtime assets.

### Current documentation layout standard

1. Keep plans in module `docs/planning/` folders.
2. Keep reports in module `docs/reports/` folders.
3. Keep checks in module `docs/checks/` folders.
4. Keep product and architecture specs in module `docs/specs/` folders.
5. Keep vision and proposal material in module `docs/vision/` folders.
